# Model Revision Changes — 2026-05-16

## Summary

These changes address the systematic review findings regarding model reproducibility,
reporting/treatment coupling, calibration enforcement, resistance validation, immunity
sensitivity, intervention decomposition, and manuscript number traceability.

## Changes Made

### 1. Reporting Rate / Treatment Rate Decoupling (Critical Fix)

**Files modified:**
- `src_python/model/ode_system.py` — Treatment rates now scaled by `diagnosis_probability` instead of `reporting_rate`
- `src_python/model/parameters.py` — Added `diagnosis_probability` field to `PreparedParameters`; defaults to baseline `reporting_rate` but is NOT modified by `reporting_multiplier`
- `src_python/simulation/run_reporting_scenarios.py` — Updated docstring to clarify observation-layer-only effect

**Effect:** Reporting-rate sensitivity scenarios now ONLY change the observation layer (how many true cases appear in surveillance). They no longer inadvertently alter treatment rates, resistance selection pressure, or transmission dynamics. The `diagnosis_probability` parameter can be independently overridden for scenarios testing changes in diagnostic access.

**Backward compatibility:** Default behavior is preserved — when no explicit `diagnosis_probability` is configured, it equals the baseline `reporting_rate` values.

### 2. Calibration Status Hard Check

**Files modified:**
- `src_python/simulation/common.py` — Added `enforce_calibration_status()` function and `require_calibrated` parameter to `run_scenario_list()`

**Effect:** Production scenario runs can now enforce that all outputs come from calibrated country profiles. If `require_calibrated=True`, the pipeline refuses to produce outputs when calibration artifacts are missing or stale.

### 3. Resistance Hindcast Validation Module (New)

**Files created:**
- `src_python/simulation/run_resistance_hindcast.py`

**Effect:** For China (2016→2025), Japan (2024→2026), and Australia (2024→2027), initializes the model at an earlier observed resistance prevalence and simulates forward across fitness values [0.85, 0.90, 0.95, 1.00, 1.05, 1.10] to check whether the model can reproduce observed trajectories. Produces scored results and an interpretation document.

### 4. Immunity Structure Sensitivity Module (New)

**Files created:**
- `src_python/simulation/run_immunity_sensitivity.py`

**Effect:** Compares four immunity structures:
1. Baseline waning-only (current model)
2. Long natural immunity (15 years — simulating strong natural boosting)
3. Short vaccine immunity (3 years — faster aP decline)
4. SIRWS-like boosting proxy (20-year natural + 3-year vaccine)

### 5. Decomposed Maternal Immunization Interventions

**Files modified:**
- `config/model_settings.yaml` — Added three new intervention scenarios:
  - `maternal_direct_antibody_only` — Passive antibody transfer only
  - `maternal_adult_boosting_only` — Adult coverage increase only
  - `maternal_cocooning_only` — Contact matrix reduction only

**Effect:** Allows attribution of the total maternal immunization effect to its constituent mechanisms.

### 6. Pipeline Orchestration (Makefile)

**Files created:**
- `Makefile`

**Effect:** Provides dependency-ordered execution: `data → calibrate → simulate → figures → validate`. Prevents stale outputs by tracking pipeline stages with stamp files.

### 7. Manuscript Text Handling

**Effect:** Manuscript text and Key Points are edited directly in the manuscript files. The pipeline no longer includes a separate manuscript-number generation stage.
